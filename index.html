Learning Analytics with Synthetic Data

Synthetic student data allows us to develop and share ideas and projects, without sharing or distributing sensitive data. That is, we share data that statistically mimics real student data, but does not contain real students! Let's see how far it can take us.

This site contains an array of tools, data, and tidbits of projects that we've assembled to aid in the development and distribution of university-level learning analytics methods and analysis. Each page addresses a separate question and includes links to the data and code that were used to create the content, all of which can downloaded and reproduced.

For those that are just entering this field, it should also give a sense of what kinds of things administrative LA data contain and how we might analyze and visualize those data using well-known tools. And don't forget: all of this code can be downloaded and run on student data by individuals on different campuses anywhere.

All of what follows makes use of the synthetic data contained in tables originally distributed with the Practical Learning Analytics MOOC. The student course and student record tables along with code are located here on Github. Most of the analysis was originally done in R, and made use of several popular R packages, such as treemap and optmatch, and most recently, optmatch. Since the initiation of the PLA-MOOC, we have constructed several new R scripts whose aim is to aid in visualizing the data contained in these tables. In many cases, R ingests, reduces, and formats the student tables, converting them into something that more sophisticated visualization tools can use. As an example, for their portability, interactivity, and awesomeness, the javascript D3js libraries are especially useful for visualzation.
